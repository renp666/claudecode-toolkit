01 什么是 Codex

OpenAI 推出的 AI 编程智能体桌面应用，对标 Claude Code，具备完整的代码库理解与操作能力。

为什么选择国产模型？

GPT-5.5 的 API 定价高达 $15-30 / 百万 Token，而国产头部模型仅需 ~$0.3-0.8 / 百万 Token，成本差距达到惊人的 30-50 倍。对于日常高频使用场景，年化成本差异可达数万元。

Codex++ 通过 CDP 注入技术实现零污染桥接，在保留 Codex 桌面版所有原生能力的前提下，无缝对接国产大模型 API，让你享受 OpenAI 的工程体验 + 国产模型的价格优势。

06 故障排查

常见问题与解决方案，涵盖配置、连接、模型行为等场景。

⚠️ 模型无响应 / 返回 404 ▼
这是最高频的配置问题。请检查：上游协议必须选择 Chat Completions，不要错误选择 Responses 协议。如果已确认协议正确，再检查 Base URL 是否正确（去掉末尾多余斜杠），以及 API Key 是否有效。
🔴 按钮灰色无法点击 ▼
通常是因为 Codex 进程没有完全退出。请完全退出 Codex（Windows 系统托盘右键退出 / macOS Cmd+Q），确认进程列表中没有 Codex 相关进程后，重新启动 Codex 和 Codex++。
🔑 配置未生效 ▼
填写完供应商配置后，必须点击"设为当前使用"按钮才能激活该配置。仅保存而不设为当前使用，Codex++ 不会路由请求到该供应商。
⏳ 连接超时 ▼
依次排查：(1) 网络是否能正常访问 API 域名 —— 用浏览器打开 Base URL 测试；(2) 是否使用代理 —— 检查系统代理设置或环境变量 HTTP_PROXY / HTTPS_PROXY；(3) API Key 是否过期或余额不足 —— 登录供应商控制台确认。
🤖 Tool Calling 不工作 ▼
不是所有国产模型都支持 function calling / tool calling。请确认你选择的模型在官方文档中明确标注支持该特性。DeepSeek V4 Pro、Qwen-Coder-Plus、GLM-4-Plus 均支持；部分 flash/turbo 版本可能不支持或能力受限。
📄 上下文过长导致异常 ▼
当模型行为开始变得不稳定（重复输出、遗漏指令、前后矛盾）时，通常是上下文接近上限。解决方案：(1) 使用 /clear 清空当前会话；(2) 使用 /context 查看实际占用；(3) 开启新会话处理新话题。建议单个会话不要超过 100K Token。
