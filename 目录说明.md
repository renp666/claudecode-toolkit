## 大类一：1-claudeCode — Claude Code 生态

### 功能一：Claude VSCode 插件安装

**目录**：`1-claudeCode\01-claude-vscode-plugin\`

**用途**：安装和配置 Claude Code 的 VSCode / Trae 插件版本

**包含文件**：
| 文件 | 说明 |
|------|------|
| `claude-code-bailian-trae-setup.md` | 百炼 Trae 平台接入指南 |

**适用场景**：
- 初次安装 Claude Code VSCode / Trae 插件
- 了解百炼 API 接入方法

### 功能二：Claude + CC-Switch 安装

**目录**：`1-claudeCode\02-claude-ccswitch\`

**用途**：安装 Claude Code CLI + CC-Switch 双版本共存方案

**包含文件**：
| 文件 | 说明 |
|------|------|
| `claude-code-cc-switch-reinstall-guide.md` | CC-Switch 安装与配置指南 |

**适用场景**：

**目录**：`2-codex\01-codex-codex++\`

**用途**：系统化介绍 Codex CLI 搭配 Codex++ 等适配工具，将国产大模型（DeepSeek、通义千问、智谱GLM等）接入 Codex 编程智能体的完整安装配置流程

**包含文件**：
| 文件 | 说明 |
|------|------|
| `CLAUDE.md` | 项目行为准则（Karpathy 四原则 + Codex 专用上下文） |
| `codex-codex++-domestic-model-setup-guide.md` | 主文档：Codex + Codex++ 国产大模型落地完整指南 |
| `常用命令.md` | Codex CLI 命令速查表（含配置速查、故障排查速查） |
| `分析-可行性评估.md` | Codex + 国产模型方案风险评估与分阶段落地建议 |
| `config-templates/*.toml` | 4 套单模型 + 1 套全量配置模板，可直接复制使用 |
| `config-templates/agents-template.md` | 通用 AGENTS.md 项目说明模板 |
| `scripts/setup-codex-deepseek.ps1` | Windows 一键配置脚本 |

**核心内容模块**：
- **前置知识**：Codex CLI 概述、国产模型适配方案三大对比
- **环境准备与基础安装**：Codex CLI 安装、Codex++ 安装、协议转换 Proxy、CC-Switch 路由
- **国产大模型配置**：DeepSeek / 通义千问 / 智谱GLM / Kimi / 其他模型的 step-by-step 配置
- **常用命令清单**：CLI 启动命令、会话内交互命令、配置管理命令
- **基础与进阶使用**：5 种基础场景 + 5 个进阶技巧 + 7 条最佳实践
- **生态资源**：协议转换工具、社区优质插件、优质 Skills 大盘点
- **故障排查**：7 个常见问题的诊断与解决方案

**适用场景**：
- 希望使用国产大模型替代/补充 OpenAI 的开发者
- 降低 AI 编程成本（DeepSeek 比 GPT-5.5 便宜 30-50x）
- 国内网络环境下稳定使用 AI 编程助手
- 多模型切换：不同任务选用不同国产模型

## 使用流程

### 第一步：初次安装
1. 阅读 `1-claudeCode\01-claude-vscode-plugin\` 中的指南，完成插件安装
2. 如需 CLI 版本，阅读 `1-claudeCode\02-claude-ccswitch\` 中的指南，安装 CC-Switch

### 第二步：环境调优
1. 阅读 `1-claudeCode\03-claude-optimize\claude调优方案.md`，了解完整调优方案
2. 运行一键部署脚本：
   ```powershell
   1-claudeCode\03-claude-optimize\scripts\setup-claude.ps1 -Phase all
   ```
3. 安装工具（可选）：
   ```powershell
   1-claudeCode\03-claude-optimize\.claude_global\tools\install-tools.ps1 -Tool all -AddToPath
   ```
4. 按需调整配置
5. 使用 `@常用命令.md` 引用到对话中快速查阅命令和快捷键

### 第三步：新项目复用
1. 将 `1-claudeCode\03-claude-optimize\templates\` 目录内容复制到新项目根目录（包含完整 .claude 结构）
2. 修改 `CLAUDE.md` 中的项目信息占位符
3. 根据项目技术栈，保留需要的 rules，删除不需要的
4. 如需 MCP 服务，确认 `.mcp.json` 中的 Token 配置
5. 从 `templates/.claude/skills/engineering/` / `misc/` / `productivity/` 按需拷贝扩展技能

### 第四步：Codex 国产大模型落地
1. 阅读 `2-codex\01-codex-codex++\codex-codex++-domestic-model-setup-guide.md`，了解完整落地方案
2. 根据使用习惯选择适配方案：Codex++（桌面App）或 协议转换Proxy（CLI）
3. 按照分模型配置指南申请 API Key 并配置 `~/.codex/config.toml`
4. 使用常用命令清单和进阶技巧提升日常使用效率

## 注意事项

1. **路径问题**：Windows 下路径使用反斜杠 `\`，PowerShell 脚本中注意转义
2. **权限问题**：部署脚本可能需要管理员权限执行（`Set-ExecutionPolicy`）
3. **环境变量**：部分工具（如 MCP 的 GitHub）需要设置对应的 Token 环境变量
4. **版本兼容**：Claude Code 和 CC-Switch 版本需匹配，详见各安装指南
5. **配置层级**：全局配置（`~/.claude/`）对所有项目生效，项目配置（`项目/.claude/`）仅对当前项目生效，两者冲突时项目级优先
6. **目录同步**：`.claude/` 与 `templates/.claude/` 需保持同步，修改一侧后记得同步另一侧
7. **settings.local.json**：用于本地个人偏好覆盖（如模型选择、权限策略），已 gitignore，不提交到仓库
