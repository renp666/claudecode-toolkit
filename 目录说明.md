## 大类一：1-claudeCode — Claude Code 生态

### 功能一：Claude + CC-Switch 安装

**目录**：`1-claudeCode\01-claude-ccswitch-安装配置\`

**用途**：安装 Claude Code CLI + CC-Switch 双版本共存方案

**包含文件**：
| 文件 | 说明 |
|------|------|
| `claude-code-CC-Switch安装指南.md` | CC-Switch 安装与配置指南 |

**适用场景**：
- 安装 Claude Code CLI 命令行版本
- 配置 CC-Switch 实现多模型路由切换
- 从 VSCode 插件迁移到 CLI 版本

**配套脚本**：`1-claudeCode\02-claude-optimize-调优方案\scripts\reinstall-claude-to-cc-switch.ps1`

## 使用流程

### 第一步：初次安装
1. 阅读 `1-claudeCode\03-claude-vscode-plugin-百炼Trae接入\` 中的指南，完成插件安装
2. 如需 CLI 版本，阅读 `1-claudeCode\01-claude-ccswitch-安装配置\` 中的指南，安装 CC-Switch

### 第二步：环境调优
1. 阅读 `1-claudeCode\02-claude-optimize-调优方案\claude调优方案.md`，了解完整调优方案
2. 运行一键部署脚本：
   ```powershell
   1-claudeCode\02-claude-optimize-调优方案\scripts\setup-claude.ps1 -Phase all
   ```
3. 安装工具（可选）：
   ```powershell
   1-claudeCode\02-claude-optimize-调优方案\01global-全局级可复制配置\tools\install-tools.ps1 -Tool all -AddToPath
   ```
4. 按需调整配置
5. 使用 `@常用命令.md` 引用到对话中快速查阅命令和快捷键

### 第三步：新项目复用
1. 将 `1-claudeCode\02-claude-optimize-调优方案\02templates-项目级可复制文件目录\` 目录内容复制到新项目根目录（包含完整 .claude 结构）
2. 修改 `CLAUDE.md` 中的项目信息占位符
3. 根据项目技术栈，保留需要的 rules，删除不需要的
4. 如需 MCP 服务，确认 `.mcp.json` 中的 Token 配置
5. 从 `02templates-项目级可复制文件目录\.claude\skills\engineering/` / `misc/` / `productivity/` 按需拷贝扩展技能

### 第四步：Codex 国产大模型落地
1. 阅读 `2-codex\01-codex-codex++\Codex++桌面版-国产大模型落地指南.md`，了解完整落地方案
2. 根据使用习惯选择适配方案：Codex++（桌面App）或 协议转换Proxy（CLI）
3. 按照指南申请 API Key 并完成配置
4. 使用常用命令清单和进阶技巧提升日常使用效率

## 注意事项

1. **路径问题**：Windows 下路径使用反斜杠 `\`，PowerShell 脚本中注意转义
2. **权限问题**：部署脚本可能需要管理员权限执行（`Set-ExecutionPolicy`）
3. **环境变量**：部分工具（如 MCP 的 GitHub）需要设置对应的 Token 环境变量
4. **版本兼容**：Claude Code 和 CC-Switch 版本需匹配，详见各安装指南
5. **配置层级**：全局配置（`~/.claude/`）对所有项目生效，项目配置（`项目/.claude/`）仅对当前项目生效，两者冲突时项目级优先
6. **目录同步**：`.claude/` 与 `02templates-项目级可复制文件目录/.claude/` 需保持同步，修改一侧后记得同步另一侧
7. **settings.local.json**：用于本地个人偏好覆盖（如模型选择、权限策略），已 gitignore，不提交到仓库
