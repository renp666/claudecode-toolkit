# Codex CLI 常用命令与快捷键速查

> 标注 ✨ 为高频使用项，建议优先掌握。标注 ⚠️ 为高风险操作。

---

## 一、CLI 启动命令

| 命令 | 作用 | 说明 |
|------|------|------|
| `codex` ✨ | 启动交互会话 | 最常用的入口，进入 Codex 编程助手 |
| `codex "提示词"` ✨ | 单次非交互执行 | 快速任务，不进入交互模式。如 `codex "解释这段代码"` |
| `codex --model qwen-coder-plus` | 指定模型启动 | 切换不同 AI 模型 |
| `codex -p deepseek-pro` | 使用预设 Profile | 不同场景切换预配置环境 |
| `codex --approval-mode auto-edit` | 自动编辑模式 | 仅批准文件编辑，命令执行仍需确认 |
| `codex --approval-mode yolo` | 全自动模式 ⚠️ | 完全自主执行，不询问确认。高风险慎用 |
| `codex --config custom.toml` | 使用指定配置文件 | 多项目多配置隔离 |
| `codex --sandbox-mode workspace-write` | 限制写入权限 | 安全审慎模式，适合审查代码 |
| `codex --verbose` | 详细输出模式 | 排查问题时使用，显示更多日志 |
| `codex --version` | 查看版本 | 确认安装状态和版本号 |
| `codex config show` | 查看当前配置 | 显示所有生效的配置项 |

## 二、会话内交互命令

### 2.1 对话管理
| 命令 | 作用 | 说明 |
|------|------|------|
| `/clear` ✨ | 清空对话 | 清除当前对话历史，开始新上下文（不删除文件） |
| `/undo` | 撤销操作 | 撤销上一步 AI 执行的文件编辑或命令 |
| `/help` | 查看帮助 | 打开帮助信息面板 |
| `/exit` 或 `Ctrl+C` | 退出会话 | 结束当前 Codex 会话 |

### 2.2 文件与变更管理
| 命令 | 作用 | 说明 |
|------|------|------|
| `/diff` ✨ | 查看待提交变更 | 展示所有 AI 修改的文件差异 |
| `/commit` | 提交变更 | AI 自动生成 commit message 并提交 |
| `/init` ✨ | 初始化项目 | 交互式问答生成 `AGENTS.md` 项目说明文件 |

### 2.3 模型与配置
| 命令 | 作用 | 说明 |
|------|------|------|
| `/model <名称>` | 切换模型 | 会话中途切换 AI 模型 |
| `/config` | 查看/编辑配置 | 进入配置管理界面 |
| `/plugins` | 浏览插件市场 | 搜索和安装 Codex 插件 |

### 2.4 插件管理
| 命令 | 作用 | 说明 |
|------|------|------|
| `codex plugin marketplace add owner/repo` | 安装插件市场 | 添加社区插件源 |
| `codex plugin install owner/repo` | 安装插件 | 从市场安装指定插件 |
| `codex plugin list` | 列出已安装插件 | 查看当前所有插件 |

## 三、交互技巧（快捷键）

### 3.1 通用快捷键
| 快捷键 | 作用 | 说明 |
|--------|------|------|
| `Ctrl + C` | 紧急中止 | 停止当前 AI 操作或代码生成 |
| `Ctrl + D` | 退出会话 | 退出 Codex CLI |
| `Tab` | 自动补全 | 输入命令或路径时按 Tab 补全 |
| `↑ / ↓` | 历史消息 | 浏览之前输入过的消息 |
| `Ctrl + L` | 清屏 | 清空终端显示 |

### 3.2 Shell 集成
| 输入方式 | 作用 | 说明 |
|----------|------|------|
| `!命令` ✨ | 执行 Shell 命令 | 如 `!git status`、`!npm test`。结果直接进入对话上下文 |
| `!!` | 重复上次 Shell | 快速再次执行上一次的 `!` 命令 |
| `\|` 管道符 | 链式处理 | 将上次结果传入新指令。如先生成代码再 `\| 写测试` |

## 四、国产模型配置速查 ✨

### 4.1 环境变量设置

```bash
# Windows PowerShell
$env:DEEPSEEK_API_KEY = "sk-你的Key"
$env:DASHSCOPE_API_KEY = "sk-你的Key"   # 通义千问
$env:ZHIPU_API_KEY = "你的Key"          # 智谱 GLM
$env:MOONSHOT_API_KEY = "sk-你的Key"    # Kimi

# macOS / Linux
export DEEPSEEK_API_KEY="sk-你的Key"
export DASHSCOPE_API_KEY="sk-你的Key"
export ZHIPU_API_KEY="你的Key"
export MOONSHOT_API_KEY="sk-你的Key"
```

### 4.2 最小配置模板（.codex/config.toml）

```toml
# DeepSeek（推荐新手首选）
model = "deepseek-v4-pro"
model_provider = "deepseek"

[model_providers.deepseek]
name = "DeepSeek"
base_url = "https://api.deepseek.com/v1"
wire_api = "responses"
env_key = "DEEPSEEK_API_KEY"
```

### 4.3 一键启动指定模型

```bash
# DeepSeek 旗舰
codex --model deepseek-v4-pro

# Qwen 编程专用
codex --model qwen-coder-plus

# GLM 免费快速
codex --model glm-4-flash

# Kimi 超长上下文
codex --model moonshot-v1-128k
```

## 五、项目配置模板速查

### 5.1 多模型 Profiles 配置

```toml
# ~/.codex/config.toml 追加以下内容
[profiles.deepseek-pro]
model = "deepseek-v4-pro"
model_provider = "deepseek"
model_reasoning_effort = "xhigh"

[profiles.qwen-fast]
model = "qwen-turbo"
model_provider = "qwen"

[profiles.glm-free]
model = "glm-4-flash"
model_provider = "glm"
```

使用：`codex -p deepseek-pro` / `codex -p qwen-fast` / `codex -p glm-free`

### 5.2 AGENTS.md 项目说明模板

```markdown
# 项目开发指南

## 技术栈
- 前端：React 19 + TypeScript + Tailwind CSS
- 后端：Node.js + Express + PostgreSQL
- 测试：Vitest + Playwright

## 编码规范
- 使用函数式组件，避免 Class
- API 错误统一返回 { error: string, code: number } 格式
- 提交信息遵循 Conventional Commits

## 常用命令
- `npm run dev`：启动开发服务器
- `npm run test`：运行测试
- `npm run lint`：代码检查
```

放在项目根目录，Codex 启动时自动加载。

## 六、故障排查速查表 ✨

| 问题 | 快速检查 | 解决命令 |
|------|---------|---------|
| Codex 找不到 | `codex --version` 是否输出版本 | `npm install -g @openai/codex` |
| API Key 不生效 | `echo $env:DEEPSEEK_API_KEY`（Win） | 重新设置环境变量 |
| 模型 404 | proxy 是否在运行 | `curl http://127.0.0.1:8787/health` |
| 配置不生效 | 检查 config.toml 语法 | `codex config show` |
| "Model metadata" 警告 | 缺少 model_properties | 添加 `[model_properties."xxx"]` 段 |
| 流式输出异常 | SSE 格式不兼容 | 检查 proxy 流式翻译开关 |
| 工具调用失败 | 模型不支持 function calling | 用 DeepSeek V4 Pro 或 Qwen-Coder-Plus |
