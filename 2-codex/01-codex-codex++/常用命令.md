# Codex++ 常用操作速查

> 标注 ✨ 为高频使用项，建议优先掌握。

---

## 一、Codex++ 管理工具操作

| 操作 | 步骤 | 说明 |
|------|------|------|
| **添加供应商** ✨ | 管理工具 → 供应商配置 → 添加供应商 | 填入名称、Base URL、Key、协议类型 |
| **切换当前供应商** ✨ | 供应商列表中选择 → 点击「当前使用」 | 切换后需通过 Codex++ 入口重新启动 |
| **会话导出** | Codex++ 会话列表 → 导出 | 支持 Markdown 格式 |

## 二、国产模型供应商配置速查 ✨

### DeepSeek（推荐首选）

| 参数 | 值 |
|------|-----|
| 名称 | `deepseek` |

## 三、Codex 对话操作

| 操作 | 方式 |
|------|------|
| 新建对话 | Codex 界面中点击新建会话 |
| 清空上下文 ✨ | 对话中输入 `/clear` |
| 撤销操作 | `/undo` 撤销上一步 |
| 查看变更 ✨ | `/diff` 查看待提交的文件修改 |
| 提交代码 | `/commit` 自动生成 commit message |
| 切换模型 | 在 Codex++ 管理工具切换供应商后重启 Codex |

## 四、AGENTS.md 项目说明模板

放在项目根目录，Codex 启动时自动加载：

```markdown
# 项目开发指南

## 技术栈
- 前端：React 19 + TypeScript + Tailwind CSS
- 后端：Node.js + Express + PostgreSQL
- 测试：Vitest + Playwright

## 编码规范
- 使用函数式组件，避免 Class
- API 错误统一返回 { error: string, code: number } 格式
- 提交信息遵循 Conventional Commits

## 常用命令
- `npm run dev`：启动开发服务器
- `npm run test`：运行测试
- `npm run lint`：代码检查
```

## 五、故障排查速查 ✨

| 问题 | 快速检查 |
|------|---------|
| 模型无响应 / 404 | 确认从 Codex++ 入口启动；检查供应商 Base URL 和 Key |
| 供应商配置不生效 | 确认已设为「当前使用」，然后重新通过 Codex++ 启动 |
| 连接超时 | 检查网络；大陆用户可尝试代理 |
| 工具调用失败 | 换用 DeepSeek V4 Pro 或 Qwen-Coder-Plus 等高能力模型 |
| 流式输出异常 | 确认供应商上游协议选择 Chat Completions |
