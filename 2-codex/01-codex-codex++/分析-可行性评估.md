# Codex++ 桌面版 + 国产大模型方案可行性评估

> 目的：评估将 Codex 桌面版通过 Codex++ 接入国产大模型的可行性、收益、风险和优先级。

---

## 一、评估结论

**可行，且生产可用。** Codex++ 社区已足够成熟（13.1K+ Stars），配合 DeepSeek / Qwen-Coder 等国产大模型，方案在 2026 年 6 月已具备生产落地条件。

推荐路径：**先通过 Codex++ 接入 DeepSeek V4 Pro 跑通 → 再按需添加其他模型供应商 → 最后引入 Skills 和插件增强。**

## 二、核心收益

| 收益维度 | 具体表现 | 量化参考 |
|----------|---------|----------|
| **成本大幅下降** | DeepSeek V4 Pro 比 GPT-5.5 便宜 30-50x | $0.5 vs $15-30 / 百万 token |
| **网络无障碍** | 无需海外支付、代理/VPN，国内直连 | 延迟 < 500ms（国内节点） |
| **中文亲和力** | 国产模型对中文项目理解更准确 | 社区反馈：中文项目 Code Review 准确率提升 |
| **多模型组合** | 高难度用 DeepSeek Pro、日常用 Qwen-Turbo | Codex++ 管理工具一键切换 |
| **零门槛配置** | Codex++ 图形化供应商管理 | 3 分钟完成首个模型接入 |

## 三、权衡与代价

| 代价 | 说明 | 应对 |
|------|------|------|
| **需要额外安装** | Codex 桌面版 + Codex++ 两个组件 | 主文档提供 step-by-step，15 分钟可完成 |
| **协议差异** | Codex 原生协议与国产模型 Chat API 不同 | Codex++ 透明转换，选择 Chat Completions 即可 |
| **工具调用兼容** | 部分模型（轻量版/免费版）对 function calling 支持不完整 | 优先用 DeepSeek V4 Pro / Qwen-Coder-Plus 等高能力模型 |
| **模型能力差异** | 国产模型在复杂多文件重构上仍有差距 | 关键任务用高能力模型，日常任务用轻量模型按需切换 |
| **API 稳定性** | 国产模型 API 偶有波动 | Codex++ 管理工具中随时切换备用供应商 |

## 四、优先级建议（分阶段落地）

| 阶段 | 内容 | 时间 | 收益确定性 |
|------|------|------|-----------|
| **Phase A** | 安装 Codex 桌面版 + Codex++ + 接入 DeepSeek | 15 分钟 | ★★★★★ |
| **Phase B** | 添加第二个模型供应商（Qwen-Coder-Plus） | 5 分钟 | ★★★★ |
