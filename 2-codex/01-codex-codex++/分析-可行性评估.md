# Codex + 国产大模型方案可行性评估

> 目的：评估将 Codex CLI 编程智能体接入国产大模型的可行性、收益、风险和优先级。

---

## 一、评估结论

**可行，且生产可用。** Codex CLI 的插件生态已足够成熟，配合社区适配工具（Codex++ / codex-proxy / CC-Switch），将国产大模型接入 Codex 的方案在 2026 年 6 月已具备生产落地条件。

推荐路径：**先用 DeepSeek V4 Pro 跑通 MVP → 再按需接入其他模型（Qwen-Coder / GLM-4）→ 最后引入 Skills 和插件增强。**

## 二、核心收益

| 收益维度 | 具体表现 | 量化参考 |
|----------|---------|----------|
| **成本大幅下降** | DeepSeek V4 Pro 比 GPT-5.5 便宜 30-50x | $0.5 vs $15-30 / 百万 token |
| **网络无障碍** | 无需海外支付、代理/VPN，国内直连 | 延迟 < 500ms（国内节点） |
| **中文亲和力** | 国产模型对中文项目理解更准确 | 社区反馈：中文项目 Code Review 准确率提升 |
| **多模型组合** | 高难度用 DeepSeek Pro、日常用 Qwen-Turbo | 按需切换，成本灵活可控 |
| **生态资源丰富** | 社区已有 280+ Codex 生态资源、13K+ Stars 的 Codex++ | Awesome Codex CLI / Skills 索引 |

## 三、权衡与代价

| 代价 | 说明 | 应对 |
|------|------|------|
| **初次配置成本** | 需要安装 2-4 个组件（Codex + 适配层 + API 平台注册） | 主文档提供 step-by-step，30 分钟可完成 |
| **协议差异** | Codex 用 Responses API，国产模型用 Chat API | 通过 Proxy 透明转换，对用户无感 |
| **工具调用兼容** | 部分模型（轻量版/免费版）对 function calling 支持不完整 | 优先用高能力模型；Proxy 自动过滤不兼容工具 |
| **模型能力差异** | 国产模型在复杂多文件重构上仍有差距 | 用 GPT-5.5 做关键任务，国产模型做常规任务 |
| **API 稳定性** | 国产模型 API 偶有 429/503 | 代理工具支持模型降级，自动切备选 |

## 四、优先级建议（分阶段落地）

| 阶段 | 内容 | 时间 | 收益确定性 |
|------|------|------|-----------|
| **Phase A** | 安装 Codex CLI + 选择一种适配方案 + 接入 DeepSeek | 30 分钟 | ★★★★★ |
| **Phase B** | 接入第二个模型（Qwen-Coder-Plus）+ 配置 Profiles 多模型切换 | 15 分钟 | ★★★★ |
| **Phase C** | 安装 Superpowers 等插件 + 配置 AGENTS.md | 20 分钟 | ★★★★ |
| **Phase D** | 编写 Agent/Context 文件 + 引入 Skills（openai/skills 社区）+ 团队共享配置 | 按需 | ★★★ |

## 五、风险清单

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 国产模型 API 配额/价格变动 | 中 | 中 | 关注官方公告，保留备用模型 |
| Codex CLI 大版本更新导致 Proxy 不兼容 | 低 | 高 | 使用社区活跃维护的 Proxy（如 codex-proxy） |
| DeepSeek 旧模型名弃用（2026/07） | 已确认 | 中 | 及时迁移到 V4 系列新模型名 |
| 网络/防火墙限制 | 低 | 低 | 使用 OpenRouter 聚合接入作为备选 |

## 六、落地验收标准

- **Phase A 验收**：能用 `codex --model deepseek-v4-pro "Hello"` 正常对话
- **Phase B 验收**：能用 `codex -p qwen-fast` 在模型间自由切换
- **Phase C 验收**：Superpowers 插件正常工作，AGENTS.md 被 Codex 正确加载
- **Phase D 验收**：团队成员可按文档 15 分钟内完成环境配置
