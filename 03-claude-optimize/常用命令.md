# Claude Code 常用命令与快捷键速查

> 标注 ✨ 为高频使用项，建议优先掌握。

---

## 一、文件与目录操作

| 输入方式 | 作用 | 说明 |
|----------|------|------|
| `@文件名` ✨ | 引用文件 | 输入 `@` 后跟文件名，弹出文件选择器，选中后文件内容会附到对话中 |
| `@目录名` | 引用目录 | 同上，选中目录会将该目录下的文件列表附到对话中 |
| `@-` | 引用最近文件 | 快速选中上一次引用的文件，无需再次搜索 |

## 二、斜杠命令（Slash Commands）

在对话中直接输入 `/命令名` 即可触发。

### 2.1 对话管理
| 命令 | 作用 | 说明 |
|------|------|------|
| `/clear` ✨ | 清空对话 | 清除当前对话历史，开始新的上下文（不会删除文件） |
| `/compact` | 压缩上下文 | 当对话很长时，压缩历史以释放 token 空间 |
| `/resume` | 恢复对话 | 恢复上一次的对话会话 |
| `/help` | 查看帮助 | 打开帮助面板 |

### 2.2 配置与权限
| 命令 | 作用 | 说明 |
|------|------|------|
| `/config` | 打开配置面板 | 图形化修改主题、模型、权限等设置 |
| `/update-config` | 修改配置文件 | 直接编辑 `settings.json`，用于配置 hooks、权限、环境变量等 |
| `/permissions` | 管理权限 | 查看和调整工具调用权限（allow / deny / ask） |
| `/add-dir` ✨ | 添加工作目录 | 将多个项目目录加入到当前会话，跨项目工作 |
| `/init` | 初始化项目 | 为新项目生成 `CLAUDE.md` 项目说明文件 |

### 2.3 代码工作流
| 命令 | 作用 | 说明 |
|------|------|------|
| `/review` | 代码审查 | 审查当前分支的代码变更 |
| `/simplify` | 代码简化 | 审查代码复用性、质量和效率，自动修复 |
| `/fix-issue` | 修复 Issue | 分析并修复 GitHub Issue |
| `/test-plan` | 测试计划 | 生成测试计划并执行 |
| `/deploy` | 部署检查 | 部署前的 checklist |
| `/security-review` | 安全审查 | 扫描安全漏洞 |

### 2.4 自动化与循环
| 命令 | 作用 | 说明 |
|------|------|------|
| `/loop` | 定时循环 ✨ | 每隔 N 分钟自动执行某个命令，如 `/loop 5m /review` |
| `/cron` | 定时任务 | 用 cron 表达式创建定时任务，支持持久化 |

### 2.5 IDE / 编辑器
| 命令 | 作用 | 说明 |
|------|------|------|
| `/ide` | 查看 IDE 状态 | 查看当前连接的 IDE（VS Code / JetBrains）状态 |
| `/doctor` | 诊断工具 | 排查 Claude Code 安装和配置问题 |

### 2.6 状态查看
| 命令 | 作用 | 说明 |
|------|------|------|
| `/status` | 查看状态 | 显示当前会话、模型、项目信息 |
| `/context` | 查看上下文 | 查看当前上下文占用情况 |
| `/tasks` | 任务列表 | 查看后台运行和计划中的任务 |
| `/terminal-setup` | 终端设置 | 配置终端集成快捷键 |

### 2.7 对话输出
| 命令 | 作用 | 说明 |
|------|------|------|
| `/export` | 导出对话 | 将当前对话导出为文件（markdown / json 等格式） |
| `/stats` | 使用统计 | 查看 Claude Code 使用统计信息 |
| `/cost` | 费用查看 | 查看 API 调用费用情况 |

## 三、交互技巧（无前缀命令）

这些不是斜杠命令，而是直接在输入框中使用的交互方式：

### 3.1 管道与重定向
| 输入方式 | 作用 | 说明 |
|----------|------|------|
| `\|` 管道符 | 将上次对话结果传入新指令 | 如：先生成代码，再输入 `\| 为这段代码写单测` |
| `!命令` ✨ | 执行 Shell 命令 | 如 `!ls` 查看文件列表、`!git diff` 查看变更。结果直接进入对话上下文 |
| `!!` | 重复上次 Shell | 快速再次执行上一次的 `!` 命令 |

### 3.2 快速操作
| 输入方式 | 作用 | 说明 |
|----------|------|------|
| `Ctrl + C` | 中止操作 | 停止当前工具调用或代码生成 |
| `Ctrl + D` | 退出会话 | 退出 Claude Code 交互 |
| `Ctrl + R` | 搜索历史 | 在对话历史中搜索 |
| `Ctrl + L` | 清屏 | 清空终端显示（等同 `/clear`） |
| `Tab` | 自动补全 | 输入命令或路径时按 Tab 补全 |
| `↑ / ↓` | 历史消息 | 浏览之前输入过的消息 |

### 3.3 图片与文件拖拽
| 输入方式 | 作用 | 说明 |
|----------|------|------|
| 拖拽图片到终端 | 上传截图 | 直接拖入图片文件，Claude 会读取并分析 |
| 拖拽文件到终端 | 上传文件 | 拖入代码文件或文档，附到对话中 |
| 粘贴图片 | 上传截图 | 直接从剪贴板粘贴截图 |

### 3.4 多行输入
| 输入方式 | 作用 | 说明 |
|----------|------|------|
| `Shift + Enter` | 换行 | 在输入框中换行，输入多行消息 |
| `Alt + Enter` | 提交多行 | 提交已输入的多行内容 |

## 四、记忆系统（Memory）

| 操作 | 说明 |
|------|------|
| "记住…" / "保存…" | 让 Claude 记住你的偏好、习惯或项目背景，持久化到 `~/.claude/projects/` 下的 memory 文件 |
| "忘记…" / "删除关于…的记忆" | 清除某条记忆 |
| "检查记忆" | 让 Claude 回顾当前保存的所有记忆 |

记忆类型：
- **user** — 你的角色、偏好、技能水平
- **project** — 项目背景、目标、里程碑
- **feedback** — 你对 Claude 输出的纠正和确认
- **reference** — 外部资源（文档链接、监控面板等）

## 六、版本控制（Git）✨

Claude Code 原生集成 Git，你只需用自然语言描述意图，AI 会自动执行对应的 Git 命令。也可以直接用 `!git` 前缀手动操作。

### 8.1 基础工作流（AI 自动执行）

| 你对 AI 说 | AI 实际做的 | 说明 |
|------------|------------|------|
| "提交代码" / "commit" | `git add` + `git commit` | 自动添加变更文件并生成规范的 commit message |
| "推送" / "push" | `git push` | 推送到远程仓库 |
| "拉取" / "pull" | `git pull` | 拉取远程最新代码 |
| "创建分支" / "new branch xxx" | `git checkout -b xxx` | 创建并切换到新分支 |
| "合并分支" / "merge xxx" | `git merge xxx` | 合并指定分支到当前分支 |
| "查看状态" / "status" | `git status` | 查看工作区变更状态 |
| "查看差异" / "diff" | `git diff` | 查看文件具体改动内容 |
| "查看日志" / "log" | `git log --oneline` | 查看提交历史 |

### 8.2 手动执行（`!` 前缀）

```bash
!git status              # 查看工作区状态
!git diff                # 查看未暂存的变更
!git diff --staged       # 查看已暂存的变更
!git add .               # 暂存所有变更
!git commit -m "..."     # 手动提交
!git push origin main    # 推送到 main 分支
!git pull origin main    # 从 main 分支拉取
!git log --oneline -10   # 查看最近 10 条提交
!git branch -a           # 查看所有分支（含远程）
!git stash               # 暂存当前修改，切换分支前常用
!git stash pop           # 恢复之前暂存的修改
```

### 8.3 分支策略

| 操作 | AI 对话示例 | 说明 |
|------|------------|------|
| 功能分支开发 | "从 main 创建 feature/login 分支" | 每个新功能独立分支 |
| 切换分支 | "切换到 main 分支" | 在分支间切换 |
| 变基 | "把当前分支 rebase 到 main" | 保持提交历史整洁（注意：会改写历史） |
| 删除分支 | "删除 feature/old 分支" | 清理已合并的旧分支 |

### 8.4 提交规范

AI 生成的 commit message 遵循 **Conventional Commits** 规范：

```
<类型>: <简短描述>

类型：
  feat     — 新功能
  fix      — 修复 bug
  refactor — 重构（不改变外部行为）
  docs     — 文档变更
  style    — 格式调整（空格、分号等）
  test     — 添加/修改测试
  chore    — 构建/工具/依赖变更
  perf     — 性能优化
```

示例：
```
feat: 添加用户登录验证

Co-Authored-By: Claude Opus 4.7 <noreply@anthropic.com>
```

### 8.5 Pull Request 工作流

| 你对 AI 说 | AI 实际做的 |
|------------|------------|
| "创建 PR" / "提交 PR" | 分析变更 → 生成标题与摘要 → 执行 `gh pr create` |
| "查看 PR" | `gh pr view` 查看 PR 详情 |
| "列出 PR" | `gh pr list` 列出所有 PR |
| "合并 PR" | `gh pr merge` 合并当前 PR |

### 8.6 安全红线 ⚠️

| 禁止行为 | 原因 |
|----------|------|
| `git push --force` 到 main/master | 会覆盖他人的提交，造成代码丢失 |
| `git reset --hard` | 不可逆地丢弃所有本地修改 |
| `git commit --amend`（已推送的提交） | 改写已发布的提交历史会导致协作混乱 |
| `git branch -D` | 强制删除未合并的分支 |
| 跳过 hooks（`--no-verify`） | 绕过代码检查和安全验证 |
| `git add -A` / `git add .`（不加确认） | 可能意外提交敏感文件（.env、密钥等） |

> **规则**：以上危险操作需要你明确授权后 AI 才会执行，不会擅自操作。

### 8.7 实用场景

```bash
# 场景1：日常开发三连
"看看改了啥"       → !git status && git diff
"提交这些改动"     → AI 暂存文件并生成 commit message
"推送到远程"       → git push

# 场景2：功能开发完整流程
"从 main 创建 feature/payment 分支"
# ... 写代码 ...
"提交并创建 PR"    → commit + push + gh pr create（一气呵成）

# 场景3：回滚误操作
"撤销刚才的提交但保留改动"  → git reset --soft HEAD~1
"丢弃所有本地改动"          → git checkout .  （⚠️ 不可逆）

# 场景4：合并冲突
"我遇到了合并冲突，帮我分析" → 读取冲突标记，给出合并建议

# 场景5：查看某人/某时间段的提交
"查看张三最近的提交"        → git log --author="张三"
"查看最近 3 天的变更"       → git log --since="3 days ago"
```

## 八、实用组合示例

```bash
# 场景1：快速审查刚写的代码
@demo.html         # 先引用你要审查的文件
/review            # 然后调用审查

# 场景2：修复 bug 前先复现
!npm test -- --grep "登录"     # 运行相关测试，结果直接进入对话
\| 根据测试报错，分析原因并修复  # 将结果传入新一轮

# 场景3：定时监控
/loop 10m !git status          # 每 10 分钟检查一次 git 状态

# 场景4：跨项目引用
/add-dir ../other-project      # 添加另一个项目
@other-project/src/utils.ts    # 引用另一个项目的文件
```
