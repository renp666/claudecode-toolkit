方案概览

四层优化架构 + 环境规划 + 目录结构，从全局到项目的完整配置体系

核心问题

Claude Code 的效率瓶颈不在模型能力，而在上下文管理：

问题表现根因
Token 浪费一次会话消耗 80K-200K Token，大部分是冗余输出CLI 命令输出未过滤、响应风格冗长
上下文腐烂长会话后 Claude 开始"犯糊涂"、矛盾、遗忘超过 120K Token 后检索准确率显著下降
需求偏差Claude 做的不是你想要的缺少实现前的需求对齐机制
无法复用每次新会话从零开始缺少跨会话记忆和项目知识沉淀

四层优化架构

架构逻辑：能用 → 做对 → 高效 → 工程化，逐层递进，每层依赖下层。

L4

工程化层 — "规模化协作"

  • 自定义 Commands：review / deploy / fix-issue，复用高频操作
  • 角色型智能体：7 个角色（UI 设计师/前端架构师/代码审查员/安全扫描器/性能分析器/API 测试/文档生成），按需调用，上下文隔离
  • Skills：mattpocock/skills + addyosmani/agent-skills，工程纪律流程
  • MCP 服务器：GitHub / Filesystem，扩展外部能力
L3

效率层 — "用更少 Token 做更多事"

  • 输出压缩：rtk（CLI 输出过滤，减少 50-90%）+ caveman（响应风格压缩，减少 65-87%）
  • 成本监控：ccusage（用量分析）+ Claude Monitor（实时预警）
  • Token 管理：/clear（切话题）、/compact（压缩）、/context（查看占用）
L2

行为约束层 — "让 Claude 做对事"

  • CLAUDE.md：全局 + 项目分层，定义行为准则
  • Rules：.claude/rules/，按模块拆分路径化规则（通用 3 个 + 技术栈 6 个）
  • Hooks：PreToolUse / PostToolUse / Stop，确定性守卫
  • 权限白名单：/permissions，减少重复确认
L1

基础层 — "能跑起来"（已就绪）

  • Claude Code CLI + CC-Switch
  • Node.js + Git + PowerShell 7
  • 网络 / 代理 / 密钥配置

环境规划

配置分为全局目录（用户级）和项目目录（项目级），各司其职。

全局目录（用户级）

放置在 ~/.claude/，放"所有项目都通用"的行为准则、常用命令、用户级设置。

~/.claude/
  CLAUDE.md                # 全局行为准则（Karpathy 四原则，38 行）
  settings.json            # 用户级设置（hooks、权限、MCP）
  commands/                # 个人常用命令（所有项目共享）
    review.md              #   /review - 代码审查
    fix-issue.md           #   /fix-issue - Issue 修复
  tools/                   # 单文件工具（rtk 等，加入 PATH）
    install-tools.ps1      #   一键下载安装工具

项目目录（项目级）

放置在 <project>/.claude/，放"只对这个项目有意义"的规则、命令、技能。可整体拷贝到新项目复用。

<project>/
  CLAUDE.md                # 项目专属上下文
  .mcp.json                # MCP 服务器配置
  .claude/
    settings.json          # 项目级设置（hooks 守卫）
    rules/                 # 路径化规则
      frontend.md          #   通用：组件规范
      backend.md           #   通用：API/数据库规范
      testing.md           #   通用：测试规范
      vue3.md              #   技术栈专用（按需）
      typescript.md        #   技术栈专用（按需）
    commands/              # 项目专属命令
      deploy.md            #   /deploy - 部署前检查
      test-plan.md         #   /test-plan - 测试计划
    agents/                # 角色型智能体（7 个）
      ui-designer.md / frontend-architect.md / code-reviewer.md
      security-scanner.md / performance-analyzer.md
      api-tester.md / doc-generator.md
    skills/                # 技能（13 个）
      caveman/ / mattpocock-tdd/ / addyosmani-spec-driven-development/
      ...

加载顺序

Claude Code 支持多级 CLAUDE.md，按作用域自动叠加加载：

  1. 先加载用户级 ~/.claude/CLAUDE.md（全局行为准则）
  2. 再加载项目级 <project>/CLAUDE.md（项目专属上下文）
  3. 两者叠加生效，项目级可覆盖全局级的重叠部分

Rules 按 glob 匹配同时生效，通用 rules 和栈专用 rules 无需删除其一。
