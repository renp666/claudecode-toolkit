快速上手

30 分钟内完成全套配置部署。支持一键脚本和手动部署两种方式。

前置条件

依赖用途安装
Node.jsnpx 安装 ccusage / MCP serversnodejs.org
Git配合 Claude Code 工作流git-scm.com
Python 3.xclaude-monitor 实时预警（可选）python.org
PowerShell 7hooks/脚本一致性（Windows 推荐）winget install Microsoft.PowerShell

方式一：一键部署（推荐）

推荐方式：使用 setup-claude.ps1 脚本，支持分阶段部署。

克隆仓库

git clone https://github.com/renp666/claudecode-toolkit.git
cd claudecode-toolkit

部署全部配置

# 部署全部三个阶段（全局 + 项目级 + 提效工具）
.\03-claude-optimize\scripts\setup-claude.ps1 -Phase all -ProjectDir "D:\your-project"

分阶段部署

# 阶段一：核心配置（CLAUDE.md + Rules + Hooks + Commands）
.\03-claude-optimize\scripts\setup-claude.ps1 -Phase 1

# 阶段二：提效增强（工具 + 监控，需先完成阶段一）
.\03-claude-optimize\scripts\setup-claude.ps1 -Phase 2 -ProjectDir "D:\your-project"

# 阶段三：高级扩展（Skills + Agents，需先完成阶段二）
.\03-claude-optimize\scripts\setup-claude.ps1 -Phase 3 -ProjectDir "D:\your-project"

安装提效工具（可选）

.\03-claude-optimize\.claude_global\tools\install-tools.ps1 -Tool all -AddToPath

方式二：手动部署

Windows (PowerShell)
macOS / Linux (bash)

阶段一：核心配置（约 30 分钟）

1

创建全局目录并部署全局配置

New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\commands" | Out-Null
New-Item -ItemType Directory -Force "$env:USERPROFILE\.claude\tools" | Out-Null
Copy-Item "03-claude-optimize\.claude_global\CLAUDE.md" "$env:USERPROFILE\.claude\CLAUDE.md"
Copy-Item "03-claude-optimize\.claude_global\commands\*.md" "$env:USERPROFILE\.claude\commands\"
2

创建项目级配置

$proj = "D:\your-project"
New-Item -ItemType Directory -Force "$proj\.claude\rules","$proj\.claude\commands","$proj\.claude\agents","$proj\.claude\skills\caveman" | Out-Null
Copy-Item "03-claude-optimize\templates\CLAUDE.md" "$proj\CLAUDE.md"
Copy-Item "03-claude-optimize\templates\.mcp.json" "$proj\.mcp.json"
Copy-Item "03-claude-optimize\.claude\rules\*.md" "$proj\.claude\rules\"
Copy-Item "03-claude-optimize\.claude\settings.json" "$proj\.claude\settings.json"
Copy-Item "03-claude-optimize\.claude\commands\*.md" "$proj\.claude\commands\"
Copy-Item "03-claude-optimize\.claude\agents\*.md" "$proj\.claude\agents\"
3

配置权限白名单

在 Claude Code 中执行 /permissions，将常用命令加入白名单。

阶段二：提效增强

4

安装 ccusage（用量监控）

npx ccusage@latest daily
5

安装 rtk（输出过滤）

从 rtk releases 下载 rtk.exe，放置到 ~/.claude/tools/rtk/ 并加入 PATH。

rtk init -g
6

安装 caveman（响应压缩）

# SKILL.md 已在本地，复制到项目
Copy-Item "03-claude-optimize\.claude\skills\caveman" "D:\your-project\.claude\skills\caveman" -Recurse

阶段三：高级扩展

7

安装 mattpocock/skills

npx skills@latest add mattpocock/skills
8

安装使用监控（可选）

pip install claude-monitor

阶段一：核心配置

1

创建全局目录并部署全局配置

mkdir -p ~/.claude/commands ~/.claude/tools
cp 03-claude-optimize/.claude_global/CLAUDE.md ~/.claude/CLAUDE.md
cp 03-claude-optimize/.claude_global/commands/*.md ~/.claude/commands/
2

创建项目级配置

proj="/path/to/your-project"
mkdir -p $proj/.claude/{rules,commands,agents,skills/caveman}
cp 03-claude-optimize/templates/CLAUDE.md $proj/CLAUDE.md
cp 03-claude-optimize/templates/.mcp.json $proj/.mcp.json
cp 03-claude-optimize/.claude/rules/*.md $proj/.claude/rules/
cp 03-claude-optimize/.claude/settings.json $proj/.claude/settings.json
cp 03-claude-optimize/.claude/commands/*.md $proj/.claude/commands/
cp 03-claude-optimize/.claude/agents/*.md $proj/.claude/agents/

阶段二、三

与 Windows 步骤相同，直接运行 npm/npx/pip 命令即可。

按需添加技术栈规则

通用 rules（frontend/backend/testing）保留为基线，按需拷贝技术栈专用规则：

# Vue 3 + TypeScript 项目
Copy-Item "03-claude-optimize\templates\.claude\rules\vue3.md" "D:\my-vue-app\.claude\rules\vue3.md"
Copy-Item "03-claude-optimize\templates\.claude\rules\typescript.md" "D:\my-vue-app\.claude\rules\typescript.md"

# Python 后端项目
Copy-Item "03-claude-optimize\templates\.claude\rules\python.md" "D:\my-api\.claude\rules\python.md"

通用 rules 和栈专用 rules 会按 glob 匹配同时生效，无需删除通用规则。

部署验证清单

阶段一验收

# 1. 全局 CLAUDE.md 存在
Get-Content "$env:USERPROFILE\.claude\CLAUDE.md" -TotalCount 5

# 2. 项目 CLAUDE.md 存在
Get-Content ".\CLAUDE.md" -TotalCount 5

# 3. Rules 目录存在
Get-ChildItem ".claude\rules"

# 4. settings.json 配置正确
Get-Content ".claude\settings.json"

# 5. 新会话自动加载上下文
claude  # 输入 /status 确认

# 6. 自定义命令可用
claude  # 输入 /review 应触发代码审查

阶段二验收

# 1. ccusage 有数据
npx ccusage@latest daily

# 2. rtk 过滤生效（运行 git status，输出应明显精简）
rtk git status

# 3. caveman 生效（输入 /caveman 后响应变为短句风格）

# 4. MCP 服务器连接正常（在 Claude 中输入 /mcp 查看状态）

阶段三验收

# 1. agents 目录有 7 个文件
Get-ChildItem ".claude\agents"

# 2. skills 目录有 13 个子目录
Get-ChildItem ".claude\skills"

# 3. 在 Claude Code 中输入 /agents，应看到自建角色列表
# 4. 调用 @code-reviewer 审查当前变更

常见使用场景

场景一：新项目初始化

# 复制 templates 到新项目
Copy-Item "03-claude-optimize\templates\*" "D:\new-project\" -Recurse
# 修改 CLAUDE.md 中的项目信息
# 按需删除不需要的 rules/agents/skills

场景二：已有项目增强

# 只添加 .claude 配置
Copy-Item "03-claude-optimize\templates\.claude" "D:\existing-project\.claude" -Recurse
# 添加项目级 CLAUDE.md
Copy-Item "03-claude-optimize\templates\CLAUDE.md" "D:\existing-project\CLAUDE.md"

场景三：全局配置更新

# 更新全局行为准则
Copy-Item "03-claude-optimize\.claude_global\CLAUDE.md" "$env:USERPROFILE\.claude\CLAUDE.md"
# 更新全局命令
Copy-Item "03-claude-optimize\.claude_global\commands\*.md" "$env:USERPROFILE\.claude\commands\"
