1. CLAUDE.md 分层配置

Claude Code 支持多级 CLAUDE.md，按作用域自动叠加加载。

第一层：全局行为准则（用户级）

放置在 ~/.claude/CLAUDE.md，只放通用行为约束，不超过 50 行。融合 Karpathy 四原则：

  1. 思考先行 — 不确定时主动提问，不假设。存在多种实现方式时列出权衡。
  2. 简洁优先 — 只做被要求的功能，不添加"灵活性"和"可扩展性"。200 行能缩到 50 行就重写。
  3. 精准变更 — 只改必须改的，不顺手"改进"旁边的代码。你的变更产生的孤立代码必须清理。
  4. 目标驱动 — 将任务转化为可验证的目标，用测试循环验证直到通过。
  5. 输出与边界 — 响应精简，不添加未被要求的注释，不引入未使用过的第三方库。
  6. 语言偏好 — 对话/注释/输出使用简体中文，commit message 保持英文。

第二层：项目专属配置（项目级）

在项目根目录创建 CLAUDE.md，只写项目专属内容，不重复全局已有的行为准则：

  • 项目信息（目标、技术栈、成功指标）
  • 目录结构说明
  • 常用命令（安装/启动/测试/构建/Lint）
  • 编码规范（语言、测试、提交格式）
  • Token 管理建议

3. Hooks 守卫

Hooks 是"保证型机制"——提示只是建议，Hook 是保证。

PreToolUse — 拦截危险命令

在 Bash 命令执行前拦截，匹配到以下模式则阻断：

  • rm -rf / — 递归删除根目录
  • git push --force — 强制推送
  • DROP TABLE — 删除数据库表
  • format [a-z]: — 格式化磁盘

PostToolUse — 文件修改提醒

Write / Edit 工具执行后，提醒运行 lint/test 确认变更质量。

Stop — 验收检查

Claude 停止响应前，提醒输出应包含：变更摘要、验证结果、风险点。

4. 权限白名单

在 Claude Code 中执行 /permissions，将常用安全命令加入白名单，避免重复确认：

  • npm run lint / npm test / npm run build
  • git status / git diff / git log
  • gh issue view / gh pr view

5. 自定义命令

自定义命令是高频操作的快捷入口，文件名即命令名。~/.claude/commands/ 全局生效，.claude/commands/ 仅项目生效。

命令级别用途可用工具
/review全局审查当前分支代码变更，按 Critical/High/Medium/Low 分类Read, Bash(git:*)
/fix-issue全局分析并修复 GitHub Issue，自动关联 commitBash(gh:*), Read, Write
/deploy项目部署前检查清单（测试/lint/环境变量/迁移/回滚）Bash(npm:*), Bash(git:*)
/test-plan项目根据需求生成结构化测试计划 + 测试文件骨架Read, Glob, Grep, Write

6. 角色型智能体（7 个）

每个角色独立 context window，不污染主对话。通过 @agent-name 显式调用或 Claude 自动委托。

🎨

ui-designer — UI 设计师

视觉一致性（颜色/间距/字体）· 组件质量（单一职责/Props 设计）· 无障碍 WCAG 2.1（alt/label/对比度/键盘导航）· 响应式适配

Read, Glob, Grep model: sonnet

🏗️

frontend-architect — 前端架构师

组件架构 · 状态管理（本地 vs 全局）· 性能（重渲染/Bundle/懒加载/虚拟滚动）· 工程规范（TS 类型/Error Boundary/路由守卫）

Read, Glob, Grep, Bash model: sonnet

🔍

code-reviewer — 代码审查员

安全漏洞（SQL注入/XSS/密钥硬编码）· 正确性（空指针/并发/资源泄漏）· 性能（N+1/循环内大对象）· 可维护性（方法过长/嵌套过深）

Read, Glob, Grep, Bash model: sonnet memory: project

🛡️

security-scanner — 安全扫描器

注入漏洞（SQL/XSS/命令/路径遍历）· 认证授权（绕过/越权/JWT 配置）· 敏感数据（日志泄露/.gitignore/加密算法）· 依赖安全

Read, Glob, Grep model: sonnet

⚡

performance-analyzer — 性能分析器

前端（Bundle 体积/首屏加载/图片优化/第三方脚本）· 后端（N+1/索引/缓存/连接池）· 资源（内存泄漏/并发/文件句柄）

Read, Glob, Grep, Bash model: sonnet

🧪

api-tester — API 测试专家

契约测试（参数类型/边界值/状态码）· 功能测试（Happy Path/边界/错误处理）· 安全测试（未认证/越权/注入）· 性能基准

Read, Glob, Grep, Write, Bash model: sonnet

📄

doc-generator — 文档生成器

API 文档（端点/参数/响应/curl 示例）· README（简介/安装/快速开始/贡献指南）· 代码注释（JSDoc/docstring）

Read, Glob, Grep, Write, Bash model: sonnet

8. MCP 服务器

通过 .mcp.json 配置，扩展 Claude Code 的外部能力。凭证通过环境变量注入，不写入仓库。

常用 MCP 服务器

  • filesystem — 文件系统操作（@modelcontextprotocol/server-filesystem）
  • github — GitHub API 集成（@modelcontextprotocol/server-github），需 GITHUB_TOKEN 环境变量

在 Claude Code 中输入 /mcp 查看连接状态。MCP 不要一次装太多，按需扩展。
